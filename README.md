# DeathRun-Server

> `ServerEngine.IOCP` 기반 C++ 멀티플레이 서버 프로토타입 (개발 중)

## 프로젝트 소개

`DeathRun-Server`는 직접 만든 `ServerEngine.IOCP` 엔진 위에서 구현 중인 게임 서버 프로젝트입니다.

처음에는 **2D 플랫포머 데스런 게임**을 목표로 기획했지만,
현재는 포트폴리오와 학습 목적에 맞춰 범위를 조정해
다음과 같은 **멀티플레이 네트워크 핵심 기능** 중심으로 개발하고 있습니다.

- 클라이언트 접속 승인
- 로비 채팅
- 로비에서 방 생성 / 참가
- 게임 플레이 중 위치 동기화
- 게임 이탈 처리

즉, 이 프로젝트는 “완성형 게임 서버”보다는
**게임 서버 개발자로서 필요한 세션/프로토콜/상태 흐름 설계 역량**을 보여주기 위한 서버 프로토타입입니다.

---

## 개발 목적

- 직접 만든 IOCP 엔진 위에 실제 게임 서버 로직을 올려보기
- 패킷 ID / 패킷 구조체 설계 경험 쌓기
- 게임 로직과 네트워크 엔진을 분리한 구조 경험
- 로비/룸/플레이 상태를 단계적으로 확장 가능한 형태로 설계하기

---

## 서버 구조

`DeathRunServerLogic`이 `IServerLogic`을 상속받아,
실제 게임 서버 로직을 담당합니다.

```text
Client
  ↓
ServerEngine.IOCP
  ├─ Listener
  ├─ SessionManager
  ├─ Session
  └─ IocpCore
        ↓
DeathRunServerLogic
```

엔진은 연결과 패킷 전달을 담당하고,
`DeathRunServerLogic`은 아래를 담당합니다.

- 접속한 세션 관리
- 패킷 ID별 로직 분기
- 채팅 브로드캐스트
- 룸/이동/이탈 관련 게임 로직 확장

---

## 현재 구현 내용

### 1. 접속 승인
클라이언트가 연결되면 서버는 세션을 등록하고,
고유 `sessionId`를 포함한 `E_ACCEPT` 패킷을 전송합니다.

이 과정을 통해 클라이언트는 서버 기준 식별자를 전달받고,
이후 룸/이동/동기화 처리의 기반으로 사용할 수 있도록 구성했습니다.

### 2. 채팅 패킷 브로드캐스트 구조
`R_CHAT` 수신 시 `S_CHAT`으로 전체 세션에 브로드캐스트하는 흐름을 구현했습니다.

이를 통해 가장 단순한 형태의 **세션 간 패킷 전파 구조**를 먼저 검증하고,
이후 룸 단위 브로드캐스트나 게임 상태 동기화로 확장하는 방향을 잡았습니다.

### 3. 게임 관련 프로토콜 설계
현재 아래 패킷 ID와 구조체를 정의해 두었습니다.

- `R_CHAT`, `S_CHAT`
- `R_JOIN`, `S_JOIN`
- `R_MOVE`, `S_MOVE`
- `E_JOIN`, `E_LEAVE`
- `N_LEAVE`
- `R_ROOM_LIST`, `S_ROOM_LIST`
- `E_ACCEPT`

또한 룸 정보, 참가 응답, 위치 동기화, 게임 이탈 등과 관련된 구조체를 설계해,
단계적으로 서버 로직을 확장할 수 있도록 준비했습니다.

## 구현 현황

| 기능 | 상태 | 설명 |
|---|---|---|
| 접속 승인 (`E_ACCEPT`) | 구현 | 서버가 세션 ID를 클라이언트에 전달 |
| 세션 목록 관리 | 구현 중 | 현재 접속 세션을 vector로 관리 |
| 채팅 패킷 처리 | 구현 | 서버 로직에서 브로드캐스트 흐름 작성 |
| 방 생성 / 참가 | 설계 완료, 로직 진행 중 | 패킷 및 구조체 정의 완료 |
| 위치 동기화 | 설계 완료, 로직 진행 중 | 패킷 ID 및 구조체 정의 완료 |
| 게임 이탈 처리 | 설계 완료, 로직 진행 중 | leave 관련 패킷 정의 완료 |
| disconnect 이후 상태 정리 | 보완 예정 | 세션/룸 정리 로직 고도화 예정 |

## 프로토콜 요약

### 채팅
- `R_CHAT` : 클라이언트 → 서버 채팅 전송
- `S_CHAT` : 서버 → 클라이언트 채팅 브로드캐스트

### 룸
- `R_JOIN` : 방 생성 또는 참가 요청
- `S_JOIN` : 방 참가 결과 및 플레이어 목록
- `R_ROOM_LIST` : 방 목록 요청
- `S_ROOM_LIST` : 현재 방 목록 응답

### 플레이 동기화
- `R_MOVE` : 내 위치 전송
- `S_MOVE` : 다른 플레이어에게 위치 전파

### 이벤트
- `E_ACCEPT` : 접속 승인 및 sessionId 전달
- `E_JOIN` : 다른 플레이어 입장 알림
- `E_LEAVE` : 다른 플레이어 퇴장 알림
- `N_LEAVE` : 자신의 게임 이탈 처리 결과

## 실행 흐름

1. `Main.cpp`에서 `DeathRunServerLogic` 생성
2. `ServerEngine` 초기화
3. 로컬 `127.0.0.1:7777` 바인딩
4. 클라이언트 접속 시 세션 생성
5. `OnConnected()` / `DispatchPacket()`에서 서버 로직 처리

## 이 프로젝트에서 강조하고 싶은 점

이 프로젝트는 “기능이 많은 게임 서버”보다,
다음 역량을 보여주는 데 초점을 두고 있습니다.

- 직접 만든 IOCP 엔진을 실제 게임 서버 로직에 적용한 경험
- 패킷 ID / 패킷 구조체를 직접 설계한 경험
- 세션 기반 멀티플레이 서버 구조 설계 경험
- 로비 → 룸 → 플레이 상태로 확장 가능한 서버 로직 구성 경험
- 엔진 코드와 게임 로직을 분리하는 설계 경험

## 회고

이 프로젝트를 진행하면서,
게임 서버 개발은 단순히 “패킷을 받아서 처리하는 일”이 아니라
**세션, 상태, 프로토콜, 흐름을 구조적으로 설계하는 일**이라는 점을 체감하고 있습니다.

현재는 개발 중인 단계지만,
엔진과 서버 로직을 분리한 상태에서 실제 멀티플레이 흐름을 구현해 나가고 있다는 점에서
제가 지향하는 **게임 서버 개발 방식**을 가장 잘 보여주는 프로젝트입니다.
